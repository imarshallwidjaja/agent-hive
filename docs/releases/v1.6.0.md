# Agent Hive v1.6.0

This release splits review responsibilities into narrower, safer agent roles and refreshes the verification path. Plan readiness, implementation quality, and technical approach advice now route to distinct reviewers, while completion evidence runs through the canonical `verification` skill.

## Added

- Reviewer roles are now split by decision type. `plan-reviewer` handles plan readiness, `code-reviewer` handles implementation diffs, and `approach-advisor` handles technical direction, architecture, and tradeoff questions.
- `approach-advisor` is available as a built-in OpenCode subagent and as a base for configured custom subagents.
- A new `verification` skill provides two explicit modes: completion gate checks before success claims, and independent verification reports for falsification-first review.

## Changed

- Primary Hive guidance now asks operators whether to consult `approach-advisor` for strategic approach questions before a plan is locked.
- Deprecated `code-reviewer`, `verification-before-completion`, and `verification-reviewer` skill bodies now act as compatibility wrappers that route agents to the canonical reviewer subagents or `verification` modes.
- Generated GitHub agents, prompts, and bundled OpenCode skills now use the split reviewer model and the canonical verification skill.
- Custom subagent configuration and schema documentation now include `approach-advisor` as a supported base agent.

## Fixed

- `hive_context_write` now returns a clear error when the requested or active feature is missing. This prevents context writes from silently creating notes for a non-existent feature after state drift or stale session recovery.
