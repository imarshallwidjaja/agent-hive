# Agent Hive v1.6.4

This patch documents the background-first orchestration contract after the runtime board, management tools, scoped launch metadata, scheduler prompts, and VS Code viewers landed.

## Added

- Documented the `hive_background_status`, `hive_background_reconcile`, `hive_background_reconcile_batch`, and `hive_background_cancel` tools for primary agents when OpenCode background subagents are enabled.
- Documented `vscode-arkive` Background Jobs and Tracked Repositories views as viewer-only state surfaces over `.hive/background-jobs.json` and `.hive/agent-hive.json`.

## Changed

- Operator docs now separate env-gate false/current blocking behavior from env-gate true/background-first scheduler behavior.
- Background docs now align with OpenCode's native completion-notification flow and explain that cancellation is not rollback.
- Background prompt acknowledgment now means the terminal result was shown once, not that the lane was reconciled into Hive state.
- Native fallback background jobs launched from task-bound parent sessions now stay feature-scoped unless a pending Hive launch explicitly provides task ownership.
- `vscode-arkive` can open non-worker background jobs at the matching `.hive/background-jobs.json` record instead of only opening the board file.
- Root docs now state that background-first orchestration adds no new agents and preserves custom derived agent inheritance.

## Compatibility

- Existing blocking task/worktree behavior remains current behavior when `OPENCODE_EXPERIMENTAL_BACKGROUND_SUBAGENTS` and `OPENCODE_EXPERIMENTAL` are unset.
- VS Code remains a review/sidebar companion. It does not cancel, reconcile, ignore, start, commit, or merge work.

## Release

- Tagged `v1.6.4` pushes publish `oc-arkive` to npm, attach `vscode-arkive.vsix` to the GitHub Release, and use this file as the release body.
