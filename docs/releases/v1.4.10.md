# Agent Hive v1.4.10

This release adds the fork-owned VS Code extension path under `vscode-arkive` and attaches a VSIX file to each GitHub Release.

## Changed

- Renamed the VS Code extension package from `vscode-hive` to `vscode-arkive` for the release VSIX artifact.
- Lowered the VS Code compatibility floor to VS Code 1.64.0 and pinned extension API typings to that floor.
- Split VS Code bundling from VSIX packaging so routine builds produce the extension bundle and release packaging produces `vscode-arkive.vsix`.
- Updated active install docs to use `oc-arkive` for OpenCode and the release-attached `vscode-arkive.vsix` for VS Code.

## Release

- The release workflow now builds and tests `hive-core`, `oc-arkive`, and `vscode-arkive`.
- Tagged releases publish `oc-arkive` to npm, attach `vscode-arkive.vsix` to the GitHub Release, and use this release note as the GitHub Release body.
- Recovery mode supports separate recovery toggles for `oc-arkive` and the GitHub Release.
