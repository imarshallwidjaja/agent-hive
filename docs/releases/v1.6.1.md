# Agent Hive v1.6.1

This release makes simplicity review a built-in Hive capability. Operators can now request a final post-implementation cleanup pass without maintaining a local custom OpenCode agent.

## Added

- `simplicity-reviewer` is now a first-class built-in read-only reviewer. It reviews completed implementation diffs for YAGNI, dead code, duplicated logic, unnecessary abstractions, redundant defensive code, and safe deletion-biased simplification.
- The reviewer is registered across Hive core configuration, OpenCode runtime agent setup, compaction re-anchoring, schema validation, generated GitHub agent artifacts, and reviewer routing tests.

## Changed

- Hive, Swarm, Architect, and Hive Builder prompts now explain when to use `simplicity-reviewer` and keep it separate from `code-reviewer`, `plan-reviewer`, `approach-advisor`, and verification flows.
- Review request guidance now supports both implementation correctness review and optional simplicity review.
- Version-bearing release surfaces now align on `1.6.1`.

## Release

- Tagged `v1.6.1` pushes publish `oc-arkive` to npm, attach `vscode-arkive.vsix` to the GitHub Release, and use this file as the release body.
