# Agent Hive v1.6.3

This patch fixes project config scoping after the repository manifest file exposed that `.hive/agent-hive.json` could mask the user's global OpenCode Hive settings.

## Changed

- Custom subagent routing guidance now describes configured custom agents as scoped specialists selected by task fit, not as default upgrades over the built-in base agents.
- Version-bearing package, lockfile, plugin manifest, changelog, and release-note surfaces now align on `1.6.3`.

## Fixed

- Project `.hive/agent-hive.json` and legacy `.opencode/agent_hive.json` files now overlay only project-scoped fields: `sandbox`, `dockerImage`, `persistentContainers`, and `repositories`.
- Global user/session policy remains sourced from `~/.config/opencode/agent_hive.json`, preserving `agentMode`, disabled MCPs, agent overrides, and custom agents when a project repository manifest exists.
- Config schema, hook cadence docs, Docker sandbox guidance, and operator instructions now describe which settings are project-scoped and which remain global-only.

## Release

- Tagged `v1.6.3` pushes publish `oc-arkive` to npm, attach `vscode-arkive.vsix` to the GitHub Release, and use this file as the release body.
