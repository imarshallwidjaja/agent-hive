# Agent Hive v1.5.1

This patch tightens the ad-hoc executor path introduced in v1.5.0. Ad-hoc merges now default to squash, optional tool inputs are normalized before they reach the worktree service, and the bundled agent guidance is clearer about parallel subagent dispatch.

## Changed

- Ad-hoc worktree integration now defaults to squash merges. Operators can still pass `strategy: "merge"` when preserving branch topology matters, or `strategy: "rebase"` when that is the intended integration path.
- `hive_adhoc_worktree_create` now treats blank `runId`, `label`, and `baseBranch` values as omitted, and treats an empty `repoIds` array as single-root mode.
- Hive, Swarm, Architect, Hive Builder, `parallel-exploration`, and `background-delegation` now describe concurrency as two separate choices: task dependency decides serial vs parallel, while wait mode decides blocking foreground vs background.

## Fixed

- Blocking `task()` usage is no longer described as a reason to serialize independent Scout fan-out. Primary agents should dispatch independent blocking tasks in one assistant message, wait for the batch, and synthesize the returned findings.
- Hive Builder guidance now tells agents to omit unused optional ad-hoc tool arguments instead of sending empty strings.

## Release

- Tagged `v1.5.1` pushes publish `oc-arkive` to npm, attach `vscode-arkive.vsix` to the GitHub Release, and use this file as the release body.
