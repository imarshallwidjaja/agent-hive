# Agent Hive v1.6.2

This release tightens custom subagent routing and hardens the release workflow after a failed `v1.6.2` tag run exposed an unchecked release-prep gap.

## Changed

- Custom derived subagents are now described as exception routes, not capability upgrades. Hive, Swarm, and planner-facing prompts should keep using the built-in base agent unless a custom agent description matches a concrete condition, or the operator explicitly names the custom agent.
- The published custom-agent example now has runtime coverage for inherited base-agent fields and custom overrides.
- Version-bearing package, lockfile, plugin manifest, changelog, and release-note surfaces now align on `1.6.2`.

## Fixed

- The `Release` workflow now validates the requested release tag against the root package version before install, build, publish, or GitHub Release steps run.
- The workflow also checks that `docs/releases/vX.Y.Z.md` exists and configures the release-notes upload with `if-no-files-found: error`, so a missing release body fails at the source instead of producing a later `release-notes` artifact download failure.

## Release

- Tagged `v1.6.2` pushes publish `oc-arkive` to npm, attach `vscode-arkive.vsix` to the GitHub Release, and use this file as the release body.
